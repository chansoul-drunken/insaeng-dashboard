2-1 · 기획
캠페인 목적·타겟 정의
"무엇을 왜 집행하는가" — 한 문장으로 확정되지 않으면 팀원과 대행사가 다른 방향으로 뛴다
목적
캠페인 방향·타겟·KPI 목표를 팀 공동 이해 상태로 만들기
트리거
업무 분류·트랙 선택 완료 후 / 기획성 업무는 월 팀 미팅 의제 상정 후 착수
핵심액션
브랜드·채널·타겟·KPI 수치 정의 → 레퍼런스 수집 → 기획서 1장 작성
완료기준
기획서 노션 저장 + 팀장 확인 완료
▼기획서 필수 항목
  • 브랜드명 (인생아구찜 / 어화락 / 삼대미역 중 선택)
  • 트랙 (B2B / B2C / PR)
  • 캠페인 목적 — 한 문장
  • 타겟 — 성별·연령·상황 1가지
  • 집행 채널 + 예산 규모
  • 목표 수치 (예: 요기요 쿠폰 집행 후 매출 +30%)
  • 레퍼런스 첨부
  • 집행 일정 (시작~종료)
▼트랙별 기획 포인트
  • B2B: 예비창업자 진입 경로(유튜브·네이버 검색) 중심. CAC 목표 선정 후 역산해서 예산 산정.
  • B2C: 배달앱 프로모션 유형(스플래시·쿠폰·서제스트) 선택 후 점주 매출 목표 수치 먼저 설정.
  • PR: 4가지 조건 중 하나 충족 여부 확인(인지도확장·내부수요·콘텐츠확산·위기대응). 미충족 시 집행 보류.
2-2 · 기획
대행사 방향 정의
집행 전 외부 파트너와 방향·기준 합의 — 브리핑이 애매하면 결과물도 애매하다
목적
집행 전 외부 파트너에게 방향·기준을 명확히 전달해 수정 횟수·낭비 시간 최소화
트리거
2-1 기획서 완성 + 대행사 집행 필요 판단 시
핵심액션
마케터 브리핑 초안 → 팀장 검토 → 대행사 전달 → 질의응답 → 납기 합의
완료기준
대행사 수신 확인 + 질문 답변 완료 + 납기일·수정 횟수 합의
▼브리핑 템플릿 필수 항목
  • 브랜드명 + 캠페인 목적
  • 타겟 정의 (성별·연령·상황)
  • 레퍼런스 2건 이상 (원하는 방향)
  • 금지사항 명시 (브랜드 이미지 훼손 표현·경쟁사 비교 등)
  • 납기일 + 수정 횟수 기준 (기본 2회)
  • 결과물 포맷·사이즈 지정
  • 집행 예산 (집행 결정된 경우)
▼대행사별 관리 포인트
대행사주요 업무주의사항
인플루언서 에이전시인생아구찜·어화락 인플루언서 섭외원본 사용 불가 확인 / 출처 밝힘 명시
배달광고사배민·요기요·쿠팡 광고 운영점주 동의 필요 쿠폰 비용 분담 사전 확인
📌 원칙: 브리핑은 마케터가 초안 작성 → 팀장이 검토하는 구조. 팀장이 직접 쓰면 팀원 역량이 늘지 않는다.

드라이브 폴더브랜드별 폴더 / 원본(.ai .psd) + 납품본(.png .jpg) 구분 저장
⚠ 주의: 구두 요청·카카오 요청은 처리하지 않는다. 폼 제출 후 접수 순서대로 처리.
